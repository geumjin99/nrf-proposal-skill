# Writing Playbook — Per-Section Tactics, Reviewer View, Rejection List, Checklist

> Source basis: 문우경 《성공적인 연구계획서 작성법》(2022), phdkim 연구제안서 팁, 대학 산학협력단
> 가이드, HiBrain/BRIC 경험. This is the "seasoned PI" knowledge: how to actually score, not just
> what sections exist.

---

## 0. The reviewer's reality (design every page around this)

> **"심사평가자가 평가를 시작한 지 보통 몇 분 만에 그 연구제안서의 수락 성패가 판가름된다."** — 문우경

- A reviewer is a **competent peer, not your sub-field specialist**, reading 5–9 similar proposals.
- Good proposal = **명확(clear) + 초점(focused) + 간결(concise)**. Bad = 산만·불명확·오타투성이.
- Many reviewers are senior and print **2-up (두 쪽 모아찍기)** → figures and key words must survive
  shrinking. Make figures **large and bold**.
- Core narrative to land in the first 1–2 pages: **중요성 + 역량/예비결과 + 추진체계 → 지원 요청.**

## 1. Per-section high-scoring tactics

### 과제명 (title) — first thing seen
- 간결·명확·분명; encode **대상 + 방법/창신점 + 목표효과**.
- **MUST match the call direction and the body** — mismatch = rejected before real review.

### 연구목표 (goal) → drives 창의성 (50)
- Avoid 광범위·대략적. Be **구체적·명확·동기 있는**, matched to 기간·연구비 (no small-money-big-claim).
- **SMART**: final goal → 연도별 정량 세부목표, each serving the final goal.
- Ambitious **and** feasible: a top reviewer comment is *"문제의식은 참신한데 이 참신성이 연구수행
  과정에 잘 반영되지 않았다."* — make sure the novelty visibly lands in §3 method.

### 연구배경·필요성 — "왜 지금 · 왜 나"
- Necessity emerges **in contrast to prior work**, not in isolation.
- Cover three axes: ① 분야/공모방향 관련성, ② 학술적 창신성(왜 지금), ③ 사회·경제적 의의.
- "왜 나" = frame the topic as something you have **continuously pursued (계속 추진해 온)** → you're
  the right executor. A short title + a tight summary lets the whole thing be read fast.

### 국내외 연구 동향/현황 + 연구공백 (gap)
- Show command of the field: make clear you reflect **the latest 동향**, not stale knowledge.
- **Research Gap = the area where missing/insufficient info blocks conclusions.** Organize prior
  work as **known / unknown / contradictory / unsolved** so the gap *points at* your goal.
- Show 역사적 맥락 — prove you're not a newcomer to the topic.
- Build this section from **live search** (Phase 1), every claim cited with a DOI/link.

### 독창성/차별성 — directly the 창의성 50
- Compare against prior work **point by point**; state "이것은 단순 반복이 아니다."
- Use a **comparison table** (선행연구 vs 본 연구: 방법/대상/관점/한계 극복) or a 개념도 so the
  difference is visible at a glance.
- 가설의 독창성 + 접근법의 차별성 are the core — but each must be executable in §3.

### 연구내용·방법 → 우수성 (30/35)
- Specify **측정도구 · 대상 · 분석방법론**; tie **each method to the 세부목표 it serves** and say
  *why* it's needed.
- **Feasibility = preliminary results.** No-paper student → strong preliminary data + lab's
  continuous work on this line. New-PhD → published-work quality + ability to run new research solo.
- **Cite literature for every method**, even routine lab methods (a top rejection cause is methods
  pasted with no citation).
- Split into clear 단계/세부목표 (e.g. 현상관찰 → 가설 → in vitro 검증 → in vivo 확증), numbered.
- A positive reviewer comment to aim for: *"선행연구 결과 기반으로 연구추진체계가 매우 구체적이고 적절."*

### 추진전략·연차별 계획 — execution credibility
- **간트차트 (Gantt) + 마일스톤 + 흐름도** is the most-expected visualization here.
- Multi-year: restructure content **by 연차 with real progression (milestones)**, not a flat split.
- 개척/발표형 programs especially probe **전략의 체계성** — show a coherent strategy, not a to-do list.

### 기대효과·활용방안 (10)
- Widen scope: **학술적 → 경제·산업적 → 사회적** 기여, concrete not hype.
- 산업화 과제: emphasize 제품화·사업화·사회적 파급효과 (weighted higher there).

### 연구비 사용계획 / 참고문헌
- 항목별 소요명세 consistent with the research content.
- 참고문헌: a curated set of **연관성 높은 최신 문헌**, one unified citation style.

## 2. Style rules

- **의의·목적 우선, 디테일은 나중.** Lead with significance, not technique dumps.
- Expand every **acronym** on first use; write for the smart outsider.
- **Visualize the core logic** with 개념도/모식도/흐름도 — "그림을 크고 분명하게."
- Emphasis (bold/italic) **only on what matters** — bolding everything kills readability.
- Problem-oriented logic: research question → 이론적 근거 → 실제 의의, each hypothesis cited.
- Avoid: 자기 하고 싶은 말만 늘어놓기 (rambling), 개념 정의만 나열 (no variable relations),
  과장 ("세상을 바꾼다" — reviewers see through it and dock points).

## 3. Rejection list — fatal flaws (use in Phase 5 simulation)

Ordered by lethality:
1. **과제명·공모방향 불일치** — dropped before real review (most fatal).
2. **분량 초과 / 양식 미준수** (font, margins) — overflow not reviewed; looks amateur.
3. **약한 도입부** — first pages don't grab → out in minutes.
4. **참신성이 수행에 반영 안 됨** — idea floats, method doesn't deliver it.
5. **막연한 연구방법 + 인용 없음**.
6. **예비결과/역량 증거 부족** — can't prove "I can do it."
7. **모호한 추진전략·연차계획** (no milestone/Gantt).
8. **과도한 전문용어/약어** — outsider reviewer can't follow.
9. **활용/적용 시각 결여** — "환자/사회에 어떻게?" unanswered.
10. **과장·허세, 산만·오타, 복잡한 가설 그림.**
11. **윤리 이슈 미확인** (생명윤리법 등).

## 4. Pre-submission checklist (Phase 6 — present to the user)

**전략·사전**
- [ ] 사업 적합성: 과제가 공모방향/사명과 부합
- [ ] 중복성 확인: 기선정·타 사업과 비중복
- [ ] 배점 인지: 창의성 50 / 우수성 30~35 / 기대효과 10 / 예산 5 에 맞춰 분량 배분

**과제명·요약**
- [ ] 과제명 = 대상+창신+목표, 본문·공모방향 일치
- [ ] 짧은 제목 + 요약문으로 3분 내 전체 파악 가능

**목표·필요성**
- [ ] SMART 목표, 기간·연구비와 매칭
- [ ] "왜 지금·왜 나"가 선행연구 대비로 드러남
- [ ] 최종목표 → 연차별 정량 세부목표 분해

**동향·차별성**
- [ ] 국내외 동향을 최신 문헌으로 정리, 연구공백 명시 (모든 인용 DOI/링크 검증)
- [ ] 선행연구 vs 본연구 비교표/개념도

**내용·방법·가능성**
- [ ] 각 방법이 어느 세부목표를 위한 것인지 명시
- [ ] 모든 방법에 참고문헌 인용
- [ ] 예비결과/연구실적으로 수행역량 입증
- [ ] 단계(현상→가설→검증→확증) 번호·단락 명확

**추진·일정**
- [ ] 간트차트 + 마일스톤 + 흐름도로 연차별 실행력 제시

**기대효과**
- [ ] 학술 → 사회·경제 순으로 확장, (산업화면) 제품화·사업화 강조

**형식·가독성**
- [ ] 분량·글자크기(≥10pt)·여백(≥15mm) 준수, 초과 없음
- [ ] 약어 첫 등장 시 풀어쓰기
- [ ] 그림 크고 분명 (두 쪽 모아찍기에도 가독)
- [ ] 강조는 핵심에만, 오타·변환오류·플레이스홀더([TODO]/[VERIFY]) 제거
- [ ] 윤리(생명윤리법 등) 자가점검
- [ ] (인문사회) 무기명 — 신원 노출 제거 완료

**시스템 (이공 2026+)**
- [ ] IRIS 회원가입 + NRI 국가연구자번호 + 최근 5년 실적 등록
- [ ] 대표 연구실적 ≤5건 / 최근 5년 / IF 나열보다 질적 우수성·계획 연관성 중심
- [ ] 마감 전 주관연구기관(기관장) 승인까지 완료 (승인 후 접수 완료, 마감 1~2일 전 권장)

**탈락 후**
- [ ] 심사 피드백 반영 후 빠르게 보완·재신청 ("아이디어 뺏기기 전에")
