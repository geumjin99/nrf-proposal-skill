# 기여 가이드 (Contributing)

이 프로젝트는 **공개·공동 제작**을 지향합니다. NRF(한국연구재단) 규정·양식·배점은 매년 바뀌므로,
여러 연구자·개발자의 손이 모일수록 정확하고 유용해집니다. 작은 수정도 환영합니다.

*This is a community project. Korean (한국어) is the primary language for issues/PRs, but English is
perfectly welcome too.*

---

## 어디에 도움이 필요한가 (Where help is most wanted)

- **사업별 평가표·배점 업데이트** — 특히 인문사회 개인연구사업 평가표 원표. 출처(신청요강 PDF/URL + 연도)를 함께.
- **HWP(.hwpx) 서식·레이아웃 충실도** — 정렬·표·이미지·공식 양식 일치도. `scripts/write_hwpx.py` 개선,
  실제 Hancom(한글)에서 연 결과 리포트.
- **검색 소스 추가/수정** — 국내(KCI·RISS·ScienceON·DBpia)·국제·시장 소스의 API/스크래핑 개선.
- **실제 사례·참고문헌 교정·번역** — 단, 아래 원칙을 지키는 범위에서.

진행 중/예정 항목은 [GitHub Issues](https://github.com/geumjin99/nrf-proposal-skill/issues)에서
확인하고 제안해 주세요.

## 지켜야 할 원칙 (Hard rules)

이 도구의 신뢰성은 아래 원칙에서 나옵니다. PR도 동일하게 따라 주세요.

1. **날조 금지.** 모든 인용은 해결 가능한 DOI/URL이 있어야 하며, 검증 불가한 것은 `[VERIFY]`로 표시.
   가짜 수치·가짜 '선정 샘플'·존재하지 않는 출처를 추가하지 않습니다.
2. **제안서 산출물에 emoji 금지.** README/문서에는 소수 허용하지만, 생성되는 계획서·템플릿·그림·스크립트
   출력에는 절대 넣지 않습니다. (`scripts/check_proposal.py`가 검출합니다.)
3. **책임 있는 사용 유지.** 이 도구는 AI 대필·자동 제출용이 아니라 **학습·참고용**이라는 입장을 약화시키지
   않습니다.
4. **규정은 출처와 함께.** NRF 규칙을 추가/수정할 때는 근거(신청요강 등)와 연도를 명시하고, 매년 바뀔 수
   있음을 전제로 작성합니다.

## 작업 흐름 (Workflow)

1. 이슈를 먼저 열어 변경 의도를 공유해 주세요(큰 변경일수록 권장).
2. 포크 → 브랜치 → 변경 → PR.
3. 스크립트를 바꿨다면 최소 한 번 실행해 동작을 확인하고, 결과를 PR 설명에 적어 주세요
   (예: `write_hwpx.py`는 `validate` 통과 + 정렬 분포 확인).
4. 韓/英 어느 언어든 좋습니다. 명확하면 충분합니다.

감사합니다 — 계획서 양식과 씨름하기보다 연구에 집중하려는 모두를 위해. 화이팅!
